Venmo User Flow Redesign Case Study

By: Elizabeth Healey

Introduction

Venmo is an app that allows people within the United States to send, receive, and transfer money instantly. Venmo is a single leg of our world that has stepped into the future while the rest of our body remains stuck in chaos. Although this app is a major advancement for a society that wants to progress our daily functions to the highest levels of ease, the frontend of the app lacks the design and functionality that the backend is serving us.

Empathize

App Analysis
One of the main features of the app is to Pay or Request money. To understand the process users go through when completing this action, I visually mapped out the current user flow.
Understanding User Needs
Thirty-two Venmo users between the ages of 18-28 years old participated in user research for this case study. They each contributed by completing a Google survey. Some users additionally contributed by answering interview questions.
93.8% or 30 participants answered they would prefer the Pay or Request button, the Transfer balance button, and their Venmo balance to be accessible from one main page.
84.4% or 27 participants expressed that the Pay or Request feature is misplaced by only being accessible on the feed page rather than on the profile page.
96.9% or 31 participants expressed that they would prefer the app to open directly to the profile page rather than the feed page.

Additionally, only 6.3% or 2 users responded that they use the feed feature which is the page users are currently directed to when opening the app.

Based on interview responses, Venmo users expressed their desired Venmo experience to be “quick, easy, and efficient.” The most used features of the app are Pay or Request, Transfer Balance, and Scan/QR code.

Defining the Problem

Pain Point 1: App Direction
The first pain point is immediate when users open the app and are directed to the feed page. Many users expressed the feed to be “unnecessary” and 93.8% or 30 individuals reported not using it. Users would prefer the app to open to the profile page where most of the main features are currently located. 96.9% or 31 individuals responded that they would prefer the app to open directly to the profile page when given the option between the profile page and feed page.
Pain Point 2: Location of the Pay or Request Feature
Unlike other main features of the app, the Pay or Request button is located on the feed page rather than the profile page. With the feed being one of the least used features of the app, users dislike having to navigate to and from the feed page in order to access the pay or request button. Users reported issues with the location of buttons being “in places that don’t make sense” and “not where they would expect.” Users also reported issues with features not being effectively “grouped together.” The location of the Pay or Request button on the feed page is disorienting, making the user flow experience less efficient and more noisy.
Design Question: How might we improve the experience of Venmo app users trying to send, request, or transfer money?

Ideate

Information Architecture
To understand the layout of features within the app, I mapped out the current information architecture.
To begin the redesign process and meet user needs, I mapped out the new information architecture I would be using.
User Flow
With the goal of improving the user flow experience when using the Pay or Request feature, I created a chart of the new user flow for the redesign.
The two main differences are in steps 1 and 2. Users will now be directed to the profile page when opening the app rather than the feed page. The Pay or Request button will be located on the profile page along with the other main app features.
Wireframes
Before the visual design, I created wireframes for both the profile page and the feed page. The user flow for the Pay or Request feature only required a redesign of the main profile page. Once the user selects the relocated Pay or Request button on the redesigned profile page, the additional processing screens were kept the same. Since I moved the Pay or Request button from the feed page to the profile page, I created a wireframe and visual design for the new feed page in addition to the new profile page. However, the feed page is no longer a part of the user flow for the Pay or Request feature.

Redesign

Based on the user research I conducted and UX design principals, I wanted to create the most organized and easy to use layout.
Redesign: Profile page
Since I was adding an additional button to the profile page, I made sure to keep the balance of the page by creating a new Pay or Request button that is equal in size to the transfer balance button, both located under the users Venmo balance. With this grouping, users can easily view and access all necessary information and features allowing faster completion of their need to send, receive, request, or transfer money. To maintain effective information hierarchy, when adding the Pay or Request button I simultaneously changed the original Transactions/Stories sliding tab to be smaller. This allowed more importance to be placed on the two main action buttons, with less importance placed on the feature that views action history.
Redesign: Feed page
Although most users in my case study reported not using the feed page, I felt it was necessary to create a redesign since I was altering the original page. By removing both the Pay or Request button and additional scan button, the page is now solely a feed page. Users no longer have to deal with the noise of other transactions when trying to access main features of the app for personal use. My decision to remove the additional Scan QR code button that previously existed on both the feed and profile page was based on user research. 81.3% or 26 individuals expressed a preference for the formatting of the circle QR code button on the profile page as opposed to the button labeled scan on the feed page. This decision was additionally connected to my goal of centering the main features on one accessible and easy to use page without unnecessary distractions. Removing misplaced features from the feed page has opened up the page making its content more clearly defined.

Pay or Request user flow redesign

By redesigning the app to open directly to the profile page along with moving the Pay or Request button to the profile page, both pain points are solved and the user flow experience for accessing all main features is improved by centering them to one page. With the focus of improving the experience of Venmo users trying to seamlessly transfer money both quickly and easily, I was able to redesign the user flow in a way that eliminates the unnecessary distractions that existed when personal use functions of the app were mixed with social media features within the app. Now when users open the app, they are brought directly to the profile page with the ability to access any main feature in one click. It is no longer necessary to navigate through multiple pages to access any main feature of the app. Once the user opens the Venmo app, they can quickly select the Pay or Request button and continue through the original processing screens to input necessary information. Once they finalize the transaction, they are brought back to the profile page where they can view their balance or use another feature. The feed page is no longer needed to access the Pay or request button and features are appropriately grouped in a way that makes sense to users.
Creating effective information hierarchy was a key component for my redesign. My intent was to guide user flow the most effectively. The two most used features of the app being Pay or Request and Transfer balance are effectively represented by being grouped together as the largest two buttons on the page. These two main features are also colored blue to add additional visual prominence. The QR code scan feature is also colored blue as well as the Friends feature. These features are sized smaller to correspond with being less used modes to transfer money but are still grouped in blue which represents action within the app.

Conclusion

This case study helped to familiarize me with the design process and it was rewarding to see all the parts come together. These redesigns would give users the quick, easy, and efficient experience they expressed a desire for. I am interested to see if Venmo will implement any user flow redesigns in the future.
